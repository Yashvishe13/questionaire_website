# Music Editing Questionnaire

Static questionnaire frontend for music edit comparisons.

## Local Use

Open `index.html` in a browser, use the auto-generated participant ID, complete the demographic survey, review the concise instructions page, and answer the 20 evaluation questions. Each question uses one original audio sample, one editing instruction, and two hidden edited candidates.

Responses are saved in browser `localStorage` until an AWS API endpoint is configured.

The participant ID is generated by the UI and is read-only. Birth month is stored with the response payload and is used with the participant ID to seed the sample assignment. The demographic payload also stores musical background.

Before each section begins, the UI opens a section introduction popup with the section definition, focus items, and ignore items from `questionnaire_draft1.md`.

At the end, participants may optionally leave an email address or Alipay account. That field is voluntary and is stored as `contactInfo` when provided. The completion screen includes the $10 Amazon gift card note.

## Study Structure

- 4 sections:
  - Harmony
  - Rhythm & Meter
  - Structural Form
  - Melodic Content & Motifs
- 5 questions per section
- 3 response options:
  - A is better
  - B is better
  - About the same
- 3 audio samples per question:
  - Original audio
  - Edited version A
  - Edited version B
- 20 questions, 20 original references, and 40 hidden edited candidates per participant

The UI does not reveal whether edited sample A or B is Librosa or MIDI. The submission payload does store `chosenEngine` for the selected candidate so the backend can save whether the participant chose Librosa or MIDI.

The current audio folder layout is:

```text
Musics/Original
Musics/Librosa
Musics/MIDI
```

The current `Original`, `Librosa`, and `MIDI` folders were created from the existing flat `Musics/*.wav` files as local placeholders. Replace those files with the real original, Librosa, and MIDI outputs when available.

## AWS-Ready Hook

Set this value in `index.html` after you create the AWS backend:

```html
<script>
  window.QUESTIONNAIRE_API_URL = "https://your-api-id.execute-api.us-east-1.amazonaws.com/prod";
</script>
```

The app will submit:

```text
POST /responses
Content-Type: application/json
```

Recommended AWS structure:

```text
Amplify Hosting or S3 + CloudFront
  -> API Gateway
  -> Lambda
  -> DynamoDB
```

You do not need AWS credentials for local development. For deployment, use an IAM user or role with permission to deploy hosting, API Gateway, Lambda, DynamoDB, IAM roles, and CloudWatch logs.
